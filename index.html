SYSTEM ONLINE — DEMO ACTIVE

A home automation & security controller, built from the sensor up.

SENTRY is a Raspberry Pi–based system that monitors motion, entry points, climate, and smoke levels, runs its own automation rules, and serves a live control dashboard — designed to run identically in simulation or on real hardware.

What it does

Six systems, one control loop

A background automation engine polls every sensor on an interval, applies rules, and reacts — the same loop that would run on a real Pi, just fed simulated data for the demo.

[ 01 ]

Motion detection

PIR sensor input triggers an alert — and a snapshot — only when the system is armed.

[ 02 ]

Entry point monitoring

Reed switch on the front door reports open/closed state and raises an alert while armed.

[ 03 ]

Climate tracking

Temperature and humidity logged continuously, with a threshold warning above 28°C.

[ 04 ]

Smoke & gas safety

Runs independently of the armed state — safety alerts never wait on user action.

[ 05 ]

Automation engine

Rule-based background thread: arming logic, dusk/dawn lighting, alert dispatch.

[ 06 ]

Live dashboard

Flask REST API + polling frontend — arm, disarm, toggle lights, watch the event feed.

Interface

Armed vs. disarmed

The status LED and control button shift from green to amber the instant the system is armed, so state is legible at a glance.

Under the hood

Built to swap from demo to hardware

Every sensor reads through one mock GPIO layer. On real hardware, a single import swap — gpio_mock → RPi.GPIO — is the only code change required.

sensors.py ──▶ automation.py ──▶ notifier.py (motion, (poll loop, (console demo, door, climate, rules, alerts) email/SMS stubs) smoke) │ │ │ ▼ ▼ gpio_mock.py database.py ──▶ Flask app.py ──▶ browser dashboard (swap for (SQLite: (REST API, (polls RPi.GPIO) events + state) /api/status) every 2.5s)
